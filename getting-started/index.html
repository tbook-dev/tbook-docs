Getting Started

Integrate TBook Vault yield into your app in under 10 minutes.

TBook Vault Gateway lets platforms offer RWA-backed USDC yield to their customers. The SDK handles all blockchain complexity — your customers just see "deposit USDC, earn yield."

Choose Your Integration Path​

PathBest ForWhat You Need
Client SDKBrowser apps with wallet adapters@tbook-dev/vault-sdk
React HooksReact apps with wallet adapter@tbook-dev/vault-react
REST APIAny backend, mobile apps, WaaS providersHTTP calls to api.tbook.com
Server SDKBackend event processing@tbook-dev/vault-node

Networks​

NetworkUse CaseUSDC
devnetTesting & developmentDevnet USDC (free from faucet)
mainnet-betaProductionReal USDC

All SDK methods and API endpoints work identically on both networks. Set the network when initializing:

// SDK
const vault = new TBookVault({ network: "devnet" });

// API
// Use tbk_test_* keys for devnet, tbk_pub_*/tbk_secret_* for mainnet

Testing on Devnet​

Use Solana devnet to test your integration end-to-end before going to production.

Devnet RPC Endpoint​

https://api.devnet.solana.com

API Keys​

Devnet API keys use a test prefix to distinguish them from production keys:

Key TypePrefixExample
Public Keypk_test_pk_test_abc123...
Secret Keysk_test_sk_test_xyz789...

Use these keys when initializing the SDK or calling the REST API on devnet.

Getting Test USDC​

You can obtain test USDC in two ways:

  1. TBook Dashboard — Request test USDC directly from the TBook Dashboard under the "Devnet Faucet" section.
  2. SPL Token Faucet — Use the spl-token-faucet CLI tool to mint devnet USDC to your wallet:
# Install the faucet tool
npm install -g @solana/spl-token-faucet

# Airdrop SOL for gas fees first
solana airdrop 2 --url https://api.devnet.solana.com

# Request test USDC
spl-token-faucet --mint USDC --amount 1000 --url https://api.devnet.solana.com

End-to-End Testing Checklist​

Follow these steps to verify your integration works correctly:

  1. Connect wallet to devnet — Configure your wallet (Phantom, Solflare, etc.) to use Solana devnet.
  2. Request test USDC from faucet — Get at least 100 test USDC using one of the methods above.
  3. Deposit test USDC into vault — Call the deposit flow through your integration and confirm the transaction succeeds on-chain.
  4. Wait for epoch to freeze and settle — The vault operator will freeze the epoch and run settlement. On devnet, this happens much faster.
  5. Verify shares received — Check that your wallet received vault shares by querying GET /users/:pubkey.
  6. Request redeem — Submit a redeem request for some or all of your shares.
  7. Claim USDC after settlement — Once the redeem epoch settles, claim your USDC back.
tip

Devnet epochs settle significantly faster than mainnet — typically in minutes rather than hours. This makes it practical to test the full deposit-settle-redeem-claim cycle in a single session.
