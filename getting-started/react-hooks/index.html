React Hooks

@tbook-dev/vault-react provides React Query–powered hooks and drop-in components over the client SDKs — positions, prices, and transaction flows without hand-rolled state management.

SDK release status

Ships at 0.2.0 with the v1 sandbox release. Surfaces shown are indicative until published; the REST contract is normative.

// Indicative 0.2.0 surface
import { TBookProvider, useAssets, usePositions, useDeposit } from "@tbook-dev/vault-react";

function Earn({ accountId }: { accountId: string }) {
  const { data: assets } = useAssets();                      // GET /v1/assets
  const { data: positions } = usePositions(accountId);      // GET /v1/accounts/:id/positions
  const deposit = useDeposit();                              // build → sign → confirm

  return (
    <button
      onClick={() => deposit.mutate({ vaultId: "rcusdp-sui", accountId, amount: "100.00" })}
    >
      Deposit
    </button>
  );
}

Pre-built components (DepositWidget, PortfolioCard, YieldBadge, TransactionHistory) cover the common "Earn tab" UI — see Use Cases for the patterns they implement.
