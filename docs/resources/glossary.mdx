# Glossary

| Term | Definition |
|------|------------|
| **Epoch** | A batch processing window in which multiple customer deposits or redemptions are collected and settled together. Epochs align on-chain settlement with off-chain RWA investment cycles. |
| **Share** | A token representing a customer's proportional ownership in the vault. Shares are minted when deposits are settled and burned when redemptions are settled. |
| **NAV** | Net Asset Value. The total value of the vault's underlying RWA portfolio. Used to calculate the share price. |
| **Share Price** | The current value of one share, calculated as NAV divided by total shares outstanding. Updated daily based on the underlying portfolio performance. |
| **Curator** | The entity responsible for managing the vault's epoch lifecycle — freezing batches, executing settlements, and updating the NAV. |
| **Platform** | The fintech application that integrates TBook Yield Layer to offer yield to its customers. Platforms control the user experience and signing keys. |
| **Customer** | An end user of the platform who deposits USDC to earn yield or redeems shares to withdraw USDC. |
| **Lazy Settlement** | An optimization where a customer's pending deposit or redemption from a previous epoch is automatically settled on their next interaction with the vault, requiring no additional transactions. |
| **USDC** | A dollar-pegged stablecoin used as the deposit and withdrawal currency in TBook vaults. |
