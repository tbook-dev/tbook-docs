# REST API

No SDK needed — the platform is fully integrable with plain HTTP from any language.

| | |
|---|---|
| Sandbox | `https://sandbox.rwa-api.tbook.com/v1` |
| Production | `https://rwa-api.tbook.com/v1` |

```bash
curl -H "Authorization: Bearer tbk_test_YOUR_KEY" \
  https://sandbox.rwa-api.tbook.com/v1/assets
```

Start here:

1. [Quickstart](/getting-started/quickstart) — the full loop in eight curl commands.
2. [Conventions](/getting-started/conventions) — errors, idempotency, pagination, amounts.
3. [API Reference](/api-reference) — every endpoint, field by field.

The only part HTTP cannot do alone is **signing**: build endpoints return unsigned transactions in `sui:tx-bytes-base64` format (the frozen legacy Solana rail uses `solana:tx-base58`), which your WaaS provider, backend keypair, or multisig signs and submits. See [Custody & WaaS Signing](/getting-started/custody-signing) for the signing recipe (Turnkey and Privy verified end-to-end) and [Transactions](/api-reference/transaction-endpoints) for the format.
