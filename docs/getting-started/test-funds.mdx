# Test Funds on Sandbox

Sandbox runs on real test networks — Sui **testnet** and Solana **devnet** — so wallets need test gas and test USDC.

## The easy way: sandbox faucet

```bash
curl -X POST https://sandbox.rwa-api.tbook.com/v1/sandbox/faucet \
  -H "Authorization: Bearer $TBK_TEST_KEY" \
  -H "Content-Type: application/json" \
  -d '{"accountId": "acct_…"}'
```

Funds test USDC and gas to the account's wallets where supported; the response reports per-chain results and links to public faucets for anything it could not fund. A **pre-funded demo account** also ships with your sandbox keys — ask your TBook contact.

## Public faucets

### Sui testnet

| What | Where |
|---|---|
| SUI gas | `sui client faucet` or the [Sui testnet faucet](https://faucet.sui.io) |
| Test USDC | Circle's testnet faucet ([faucet.circle.com](https://faucet.circle.com)) — select Sui testnet |

### Solana devnet

| What | Where |
|---|---|
| SOL gas | `solana airdrop 2 --url devnet` |
| Test USDC | [faucet.circle.com](https://faucet.circle.com) — select Solana devnet |

:::note
Use the **canonical Circle test USDC** mints — the exact token types accepted by each vault are listed in the vault's `addresses` in [`GET /v1/vaults/:vaultId`](/api-reference/vault-endpoints). Tokens from third-party faucets with different mints will not work.
:::

## Settlement on sandbox

Settlement cycles run **daily** on sandbox, and on demand via [`POST /v1/sandbox/settle`](/api-reference/sandbox) — so you can test the full deposit → settle → redeem → claim loop in minutes instead of waiting for T+1.
