# Transaction Endpoints

All POST endpoints return an unsigned transaction. Sign it with your wallet or WaaS provider and submit to the network.

---

## POST /v1/vaults/:vaultId/tx/deposit

Build a deposit transaction.

**Path params:** `vaultId` — Vault identifier

**Body:**

```json
{
  "user": "WALLET_PUBKEY",
  "amountUsdc": 100
}
```

**Response:**

```json
{
  "transaction": "base58_encoded_unsigned_transaction",
  "message": "Deposit 100 USDC into TBook Vault (Epoch #2)"
}
```

---

## POST /v1/vaults/:vaultId/tx/cancel-deposit

Cancel a pending deposit (0.1% fee applies).

**Path params:** `vaultId` — Vault identifier

**Body:**

```json
{
  "user": "WALLET_PUBKEY"
}
```

**Response:**

```json
{
  "transaction": "base58_encoded_unsigned_transaction",
  "message": "Cancel pending deposit (0.1% fee)"
}
```

:::info
Cancellation is only possible while the deposit epoch is in the **Open** state. Once the epoch is frozen, deposits cannot be cancelled.
:::
