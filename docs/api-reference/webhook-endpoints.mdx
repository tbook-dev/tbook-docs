# Webhook Management

Manage webhook endpoints and read the event log. Secret key required. For payloads, signatures, and retry behavior, see the [Webhooks guide](/webhooks).

---

## POST /v1/webhooks

**Body:**

| Field | Notes |
|---|---|
| `url` | HTTPS only. Must resolve to a public address — private, link-local, and cloud-metadata IP ranges are rejected (`invalid_request`). |
| `events[]` | Event names or wildcards: `["transaction.*", "vault.*"]`. See the [event catalog](/webhooks/event-types). |

```json
{
  "id": "wh_01h…",
  "url": "https://your-server.example/tbook",
  "events": ["transaction.*", "vault.*"],
  "secret": "whsec_…"
}
```

**The `secret` is shown once.** Store it like a password — it verifies every delivery. Max 10 endpoints per organization.

## GET /v1/webhooks

List endpoints (secrets masked).

## PATCH /v1/webhooks/:id

Update `url` and/or `events` — include only what changes.

## DELETE /v1/webhooks/:id

Stop deliveries immediately.

## POST /v1/webhooks/:id/rotate-secret

Returns a new secret. The old secret stays valid for **24 hours in parallel** so you can roll your servers without dropping verification.

---

## GET /v1/events

Your organization's event log — also the **polling fallback** if your endpoint was down past the retry window.

**Query:** `type=`, `from=`, `limit=`, `cursor=`.

```json
{
  "data": [
    {
      "id": "evt_01h…",
      "type": "transaction.deposit.settled",
      "created": "2026-06-15T08:14:05Z",
      "data": { "intentId": "txi_…", "accountId": "acct_…", "amount": "100.00" },
      "deliveries": [
        { "webhookId": "wh_01h…", "status": "delivered", "attempts": 1,
          "lastAttemptAt": "2026-06-15T08:14:06Z" }
      ]
    }
  ],
  "nextCursor": null, "hasMore": false
}
```

`deliveries[].status`: `delivered` · `retrying` · `failed`. Consume idempotently keyed on `event.id` — the same event can be delivered more than once.
