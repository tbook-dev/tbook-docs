Event Catalog

All events share the envelope {id, type, created, data}. Per-account events include accountId and your externalId.

Transaction events (per account)​

transaction.deposit.confirmed​

Deposit transaction observed on-chain. Funds are in the current settlement cycle.

{
  "id": "evt_01h…", "type": "transaction.deposit.confirmed", "created": "…",
  "data": {
    "intentId": "txi_01h…", "accountId": "acct_7f3a…", "externalId": "user-8492",
    "vaultId": "rcusdp-sui", "amount": "100.00", "digest": "8vJk…",
    "settlementCycle": 6
  }
}

Suggested UI: "Processing — settles T+1."

transaction.deposit.settled​

Shares credited.

{
  "type": "transaction.deposit.settled",
  "data": {
    "intentId": "txi_01h…", "accountId": "acct_7f3a…", "externalId": "user-8492",
    "vaultId": "rcusdp-sui", "amount": "100.00",
    "shares": "97.31", "sharePrice": "1.0275", "settlementCycle": 6
  }
}

Suggested UI: "Your deposit is now earning." Omnibus: this is your booking trigger — allocate shares in your sub-ledger at sharePrice.

transaction.deposit.cancelled​

User cancel or cycle rollback (reason: "user_cancel" | "rollback"). On rollback the amount is back in the wallet (direct rail) or claimable (gateway rail). Omnibus: reverse the pending allocation in your ledger.

transaction.redeem.confirmed / transaction.redeem.settled​

Redemption queued / proceeds claimable. settled data includes claimable (USDC) and settlementCycle.

Suggested UI on settled: "Withdrawal ready" with a claim action.

transaction.claim.confirmed​

USDC paid to the account's own wallet. Data: amount, digest.

Vault events (organization-wide)​

vault.cycle.closed / vault.cycle.settled / vault.cycle.rolled_back​

A settlement cycle changed state. Data: vaultId, settlementCycle, totals (totalDeposits, totalRedeems, and on settled the final sharePrice).

Per-account transaction.*.settled events follow vault.cycle.settled for every affected account — most apps only need the per-account events; the cycle events are useful for ops dashboards and omnibus reconciliation.

vault.price.updated​

NAV changed. Data: vaultId, price, apy, asOf.

vault.paused / vault.unpaused​

Disable/enable deposit & redeem actions in your UI. Claims keep working while paused.

Account & treasury events​

TypeFires whenKey data fields
account.createdaccount registeredaccountId, externalId, type
treasury.allowlist.activated24 h timelock elapsedid, accountId, address, chain
treasury.withdrawal.pending_approvalwithdrawal awaits a second keyid, accountId, amount
treasury.withdrawal.approvedapproved; unsigned tx availableid, intentId
treasury.withdrawal.confirmedtransfer on-chainid, amount, address, digest

Wildcards​

Subscriptions accept wildcards: transaction.*, vault.*, treasury.*, or * for everything. New event types may be added under existing prefixes at any time — handle unknown types gracefully.
