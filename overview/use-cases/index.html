Use Cases

How fintech platforms integrate TBook Yield Layer to offer yield to their customers. Each use case includes recommended UI patterns (based on the vault.tbook.com reference implementation) and the API calls needed to implement them.

Multi-Vault Architecture

TBook Yield Layer supports multiple vaults, each backed by a different RWA strategy (T-Bills, Private Credit, Gold, etc.). All API calls are scoped to a specific vaultId. Your platform can offer one vault or multiple — giving customers a choice of risk/return profiles.

1. Earn Tab (Consumer-Facing)​

Scenario: Your app has an "Earn" section where end-users browse available yield strategies, deposit USDC, and see yield accruing in real time.

Recommended UI Pattern​

Vault Selector: Show a list of available vaults, each displaying its name, current APY, and total value locked. Users pick a vault, then interact with deposit/redeem tabs scoped to that vault.

Balance Breakdown (per vault):

Balance SectionDescription
AccruingShares currently earning yield (shares × share price)
PendingDeposits submitted but not yet settled into shares
RedeemingShares submitted for redemption, awaiting settlement
ClaimableUSDC ready to withdraw after redeem settlement

Include an APY chart per vault showing historical yield performance, and display estimated daily earnings based on the user's current balance and the latest APY.

    const { priceNum, apy } = await res.json();
    return { vaultId, sharePrice: priceNum, apy };

);

2. Treasury Management (Platform-Facing)​

Scenario: Your fintech platform holds idle USDC (customer float, reserves, or operational funds). Deploy it across multiple vault strategies to optimize risk-adjusted returns. No end-user interaction needed — this runs entirely on your backend.

Recommended UI Pattern​

Build an admin dashboard (internal only) that displays:

  • Vault Allocation Table — breakdown by vault: TVL, APY, allocation percentage
  • Total Yield Earned — cumulative earnings across all vaults
  • Epoch Status — current epoch state per vault (Accruing / Frozen / Settled)
  • Rebalance Controls — move funds between vaults as strategy needs change

API Integration​

Deploy treasury across multiple vaults

// Allocation strategy: 60% T-Bills (safe), 30% Private Credit (growth), 10% Gold (hedge)
const allocations = [
  { vaultId: "tbills-01", amountUsdc: 300_000 },
  { vaultId: "private-credit-01", amountUsdc: 150_000 },
  { vaultId: "gold-01", amountUsdc: 50_000 },
];

for (const { vaultId, amountUsdc } of allocations) {
  const txResponse = await fetch(
    `https://api.tbook.com/v1/vaults/${vaultId}/tx/deposit`,
    {
      method: "POST",
      headers: {
        "x-api-key": "sk_test_YOUR_SECRET_KEY",
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        user: TREASURY_WALLET_PUBKEY,
        amountUsdc,
      }),
    }
  );
  const { transaction } = await txResponse.json();
  const signed = signWithKeypair(transaction, treasuryKeypair);
  await connection.sendRawTransaction(signed);
}

Monitor all vaults via webhook

// Webhook handler: POST /webhooks/tbook
app.post("/webhooks/tbook", (req, res) => {
  const event = req.body;

  if (event.type === "deposit_epoch.settled") {
    const { vaultId, epochId, sharesMinted, sharePrice } = event.data;
    // Update internal accounting per vault
    updateTreasuryLedger(vaultId, sharesMinted, sharePrice);
    console.log(
      `[${vaultId}] Epoch ${epochId}: ${sharesMinted} shares at $${sharePrice}`
    );
  }

  if (event.type === "redeem_epoch.settled") {
    const { vaultId, epochId, usdcReturned } = event.data;
    // Trigger claim to pull USDC back to treasury
    triggerClaim(vaultId, TREASURY_WALLET_PUBKEY);
    console.log(`[${vaultId}] Redeem epoch ${epochId}: $${usdcReturned} ready`);
  }

  res.sendStatus(200);
});

Dashboard: aggregate positions across vaults

// Fetch treasury positions across all vaults
const vaultIds = ["tbills-01", "private-credit-01", "gold-01"];

const dashboard = await Promise.all(
  vaultIds.map(async (vaultId) => {

    return {
      vaultId,
      vault: await vaultRes.json(),

3. Savings Account (White-Label)​

Scenario: Wrap TBook Vault as a "High-Yield Savings Account" in your consumer app. Users see their balance grow in USD terms — they never interact with shares, epochs, or blockchain concepts. Optionally back the savings product with multiple vaults for blended yield.

Recommended UI Pattern​

Show users a simple savings balance in USD that grows over time:

  • Display portfolioValueUsdc as the "Savings Balance" (not share count)
  • Show "Interest Earned" as the difference between total deposits and current portfolio value
  • Use a line chart showing balance growth over the past 30/90 days
  • Hide all blockchain terminology (shares, epochs, settlement)

API Integration​

Display USD balance (single vault or blended)

// Option A: Single vault (simplest)
const vaultId = "tbills-01";

const [userRes, priceRes] = await Promise.all([
  fetch(`https://api.tbook.com/v1/vaults/${vaultId}/users/${userPubkey}`, {
    headers: { "x-api-key": "pk_test_YOUR_KEY" },
  }),
  fetch(`https://api.tbook.com/v1/vaults/${vaultId}/price`, {
    headers: { "x-api-key": "pk_test_YOUR_KEY" },
  }),
]);
const { portfolioValueUsdc } = await userRes.json();
const { apy } = await priceRes.json();

const estimatedDailyYield = (portfolioValueUsdc * parseFloat(apy)) / 100 / 365;
// "Your savings: $10,247.83"
// "Earning ~$1.35/day"
// Option B: Blended across multiple vaults (higher yield, diversified)
const vaultIds = ["tbills-01", "private-credit-01"];

let totalBalance = 0;
let weightedApy = 0;

for (const vaultId of vaultIds) {
  const [userRes, priceRes] = await Promise.all([
    fetch(`https://api.tbook.com/v1/vaults/${vaultId}/users/${userPubkey}`, {
      headers: { "x-api-key": "pk_test_YOUR_KEY" },
    }),
    fetch(`https://api.tbook.com/v1/vaults/${vaultId}/price`, {
      headers: { "x-api-key": "pk_test_YOUR_KEY" },
    }),
  ]);
  const { portfolioValueUsdc } = await userRes.json();
  const { apy } = await priceRes.json();
  totalBalance += portfolioValueUsdc;
  weightedApy += portfolioValueUsdc * parseFloat(apy);
}

const blendedApy = weightedApy / totalBalance;
// "Your savings: $15,272.33"
// "Blended APY: 6.49%"

4. Card-Linked Yield​

Scenario: Users fund a prepaid card account. Idle funds automatically earn yield in the background. When the user spends, the platform automatically redeems the required amount from the vault to cover the charge.

Recommended UI Pattern​

  • Card balance shows total available (vault shares value + any un-deposited USDC)
  • Yield earned badge shows cumulative interest earned on the card balance
  • Transaction history mixes card spends with yield accrual entries
  • All vault mechanics are invisible to the user

API Integration​

Fund card (auto-deposit to vault)

// Platform selects the vault strategy for card funds
const vaultId = "tbills-01"; // T-Bills: safe, liquid, ideal for card balances

async function fundCard(userPubkey: string, amountUsdc: number) {
  const txResponse = await fetch(
    `https://api.tbook.com/v1/vaults/${vaultId}/tx/deposit`,
    {
      method: "POST",
      headers: {
        "x-api-key": "sk_test_YOUR_SECRET_KEY",
        "Content-Type": "application/json",
      },
      body: JSON.stringify({ user: userPubkey, amountUsdc }),
    }
  );
  const { transaction } = await txResponse.json();
  return submitTransaction(transaction);
}

Spend (auto-redeem from vault)

async function processCardSpend(userPubkey: string, spendAmountUsdc: number) {
  const vaultId = "tbills-01";

  // Step 1: Get current share price to calculate shares needed
  const priceResponse = await fetch(
    `https://api.tbook.com/v1/vaults/${vaultId}/price`,
    { headers: { "x-api-key": "sk_test_YOUR_SECRET_KEY" } }
  );
  const { priceNum } = await priceResponse.json();
  const sharesNeeded = Math.ceil(spendAmountUsdc / priceNum);

  // Step 2: Redeem shares
  await fetch(`https://api.tbook.com/v1/vaults/${vaultId}/tx/redeem`, {
    method: "POST",
    headers: {
      "x-api-key": "sk_test_YOUR_SECRET_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      user: userPubkey,
      shares: sharesNeeded,
    }),
  });

  // Step 3: After settlement, claim USDC and process card payment
  await fetch(`https://api.tbook.com/v1/vaults/${vaultId}/tx/claim`, {
    method: "POST",
    headers: {
      "x-api-key": "sk_test_YOUR_SECRET_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ user: userPubkey }),
  });
}

Listen for settlement to complete card transactions

app.post("/webhooks/tbook", (req, res) => {
  const event = req.body;

  if (event.type === "redeem_epoch.settled") {
    const { vaultId, userPubkey, usdcReturned } = event.data;
    // Release funds to card network for the pending charge
    finalizeCardPayment(userPubkey, usdcReturned);
    console.log(
      `[${vaultId}] Card spend settled: $${usdcReturned} for ${userPubkey}`
    );
  }

  res.sendStatus(200);
});
