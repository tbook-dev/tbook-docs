Use Cases

How fintech platforms integrate TBook Vault to offer yield to their customers. Each use case includes recommended UI patterns (based on the vault.tbook.com reference implementation) and the API calls needed to implement them.

1. Earn Tab (Consumer-Facing)​

Scenario: Your app has an "Earn" section where end-users deposit USDC and see yield accruing in real time. Users see a simple "Earn X% APY on USDC" card and can deposit or redeem with one tap.

Recommended UI Pattern​

Build a tabbed interface with Deposit and Redeem tabs. Below the action area, show a balance breakdown:

Balance SectionDescription
AccruingShares currently earning yield (shares x share price)
PendingDeposits submitted but not yet settled into shares
RedeemingShares submitted for redemption, awaiting settlement
ClaimableUSDC ready to withdraw after redeem settlement

Include an APY chart showing historical yield performance, and display estimated daily earnings based on the user's current balance and the latest APY.

API Integration​

Step 1: Display current APY and share price

// Fetch current vault pricing
const response = await fetch("https://api.tbook.com/v1/price", {
  headers: { "x-api-key": "pk_test_YOUR_KEY" },
});
const { sharePrice, apy, totalValueLocked } = await response.json();

// Display to user
// "Earn {apy}% APY on USDC"
// "1 share = {sharePrice} USDC"

Step 2: Create a deposit transaction

// Build the deposit transaction
const txResponse = await fetch("https://api.tbook.com/v1/tx/deposit", {
  method: "POST",
  headers: {
    "x-api-key": "pk_test_YOUR_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    userPubkey: wallet.publicKey.toBase58(),
    amountUsdc: 100, // deposit 100 USDC
  }),
});
const { transaction } = await txResponse.json();

// Sign and submit via wallet adapter
const signed = await wallet.signTransaction(transaction);
const txHash = await connection.sendRawTransaction(signed.serialize());

Step 3: Show user balances

// Fetch user portfolio
const userResponse = await fetch(
  `https://api.tbook.com/v1/users/${wallet.publicKey.toBase58()}`,
  { headers: { "x-api-key": "pk_test_YOUR_KEY" } }
);
const {
  sharesBalance,
  portfolioValueUsdc,
  pendingDepositsUsdc,
  redeemingUsdc,
  claimableUsdc,
} = await userResponse.json();

2. Treasury Management (Platform-Facing)​

Scenario: Your fintech platform holds idle USDC (customer float, reserves, or operational funds). Instead of letting it sit, deploy it into T-Bill-backed yield strategies. No end-user interaction is needed — this runs entirely on your backend.

Recommended UI Pattern​

Build an admin dashboard (internal only) that displays:

  • Total Value Locked (TVL) — total USDC deployed into the vault
  • Current APY — live yield rate
  • Total Yield Earned — cumulative earnings since deployment
  • Epoch Status — current epoch state (Accruing / Frozen / Settled)

API Integration​

Deposit idle treasury funds (backend)

// Server-side: deposit platform treasury
const txResponse = await fetch("https://api.tbook.com/v1/tx/deposit", {
  method: "POST",
  headers: {
    "x-api-key": "sk_test_YOUR_SECRET_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    userPubkey: TREASURY_WALLET_PUBKEY,
    amountUsdc: 500_000, // deploy $500k
  }),
});
const { transaction } = await txResponse.json();

// Sign with treasury wallet keypair and submit
const signed = signWithKeypair(transaction, treasuryKeypair);
const txHash = await connection.sendRawTransaction(signed);

Listen for settlement events (webhook)

// Webhook handler: POST /webhooks/tbook
app.post("/webhooks/tbook", (req, res) => {
  const event = req.body;

  if (event.type === "deposit_epoch.settled") {
    const { epochId, sharesMinted, sharePrice } = event.data;
    // Update internal accounting
    // Record: sharesMinted shares at sharePrice per share
    console.log(
      `Epoch ${epochId} settled: ${sharesMinted} shares at $${sharePrice}`
    );
  }

  if (event.type === "redeem_epoch.settled") {
    const { epochId, usdcReturned } = event.data;
    // Trigger claim to pull USDC back to treasury
    console.log(`Redeem epoch ${epochId}: $${usdcReturned} ready to claim`);
  }

  res.sendStatus(200);
});

3. Savings Account (White-Label)​

Scenario: Wrap TBook Vault as a "High-Yield Savings Account" in your consumer app. Users see their balance grow in USD terms — they never interact with shares, epochs, or blockchain concepts.

Recommended UI Pattern​

Show users a simple savings balance in USD that grows over time:

  • Display portfolioValueUsdc as the "Savings Balance" (not share count)
  • Show "Interest Earned" as the difference between total deposits and current portfolio value
  • Use a line chart showing balance growth over the past 30/90 days
  • Hide all blockchain terminology (shares, epochs, settlement)

Key Implementation Detail: Your backend converts shares x share_price into a USD balance for display. Users never see share counts.

API Integration​

Display USD balance (hide shares)

// Fetch user's portfolio value in USDC
const userResponse = await fetch(
  `https://api.tbook.com/v1/users/${userPubkey}`,
  { headers: { "x-api-key": "pk_test_YOUR_KEY" } }
);
const { portfolioValueUsdc, sharesBalance } = await userResponse.json();

// Fetch current share price
const priceResponse = await fetch("https://api.tbook.com/v1/price", {
  headers: { "x-api-key": "pk_test_YOUR_KEY" },
});
const { sharePrice, apy } = await priceResponse.json();

// Display to user in USD terms only
const displayBalance = portfolioValueUsdc; // e.g., "$10,247.83"
const estimatedDailyYield = (portfolioValueUsdc * apy) / 100 / 365;
// "Your savings: $10,247.83"
// "Earning ~$1.40/day"

Deposit (presented as "Add to Savings")

// User taps "Add to Savings" — internally this creates a vault deposit
const txResponse = await fetch("https://api.tbook.com/v1/tx/deposit", {
  method: "POST",
  headers: {
    "x-api-key": "pk_test_YOUR_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    userPubkey: userPubkey,
    amountUsdc: depositAmount,
  }),
});

Withdraw (presented as "Withdraw from Savings")

// User taps "Withdraw" — internally this redeems shares then claims USDC
// Step 1: Redeem shares
const redeemResponse = await fetch("https://api.tbook.com/v1/tx/redeem", {
  method: "POST",
  headers: {
    "x-api-key": "pk_test_YOUR_KEY",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    userPubkey: userPubkey,
    amountShares: sharesToRedeem,
  }),
});

// Step 2: After epoch settles, claim USDC

  method: "POST",
  headers: {

    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    userPubkey: userPubkey,

4. Card-Linked Yield​

Scenario: Users fund a prepaid card account. Idle funds automatically earn yield in the background. When the user spends, the platform automatically redeems the required amount from the vault to cover the charge.

Recommended UI Pattern​

  • Card balance shows total available (vault shares value + any un-deposited USDC)
  • Yield earned badge shows cumulative interest earned on the card balance
  • Transaction history mixes card spends with yield accrual entries
  • All vault mechanics are invisible to the user

API Integration​

Fund card (auto-deposit to vault)

// When user loads funds onto their card, deposit into vault
async function fundCard(userPubkey: string, amountUsdc: number) {
  const txResponse = await fetch("https://api.tbook.com/v1/tx/deposit", {
    method: "POST",
    headers: {
      "x-api-key": "sk_test_YOUR_SECRET_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ userPubkey, amountUsdc }),
  });
  const { transaction } = await txResponse.json();
  // Sign and submit
  return submitTransaction(transaction);
}

Spend (auto-redeem from vault)

// When a card charge comes in, redeem shares to cover it
async function processCardSpend(userPubkey: string, spendAmountUsdc: number) {
  // Step 1: Calculate shares needed
  const priceResponse = await fetch("https://api.tbook.com/v1/price", {
    headers: { "x-api-key": "sk_test_YOUR_SECRET_KEY" },
  });
  const { sharePrice } = await priceResponse.json();
  const sharesNeeded = Math.ceil(spendAmountUsdc / sharePrice);

  // Step 2: Redeem shares
  const redeemResponse = await fetch("https://api.tbook.com/v1/tx/redeem", {
    method: "POST",
    headers: {
      "x-api-key": "sk_test_YOUR_SECRET_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      userPubkey,
      amountShares: sharesNeeded,
    }),
  });

  // Step 3: After settlement, claim USDC and process card payment
  const claimResponse = await fetch("https://api.tbook.com/v1/tx/claim", {
    method: "POST",
    headers: {
      "x-api-key": "sk_test_YOUR_SECRET_KEY",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ userPubkey }),
  });

  return claimResponse.json();
}

Listen for settlement to complete card transactions

// Webhook: finalize card spend after redeem settles
app.post("/webhooks/tbook", (req, res) => {
  const event = req.body;

  if (event.type === "redeem_epoch.settled") {
    const { userPubkey, usdcReturned } = event.data;
    // Release funds to card network for the pending charge
    finalizeCardPayment(userPubkey, usdcReturned);
  }

  res.sendStatus(200);
});
